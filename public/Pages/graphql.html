GraphQL with HotChocolate

What is GraphQL?

GraphQL is a query language for APIs and a runtime for executing those queries with your existing data. It allows clients to request exactly the data they need, improving performance and reducing over-fetching.

Setting Up HotChocolate

HotChocolate is a .NET GraphQL server implementation that supports both schema-first and code-first approaches. To set up HotChocolate in your project, follow these steps:


1. Install the HotChocolate NuGet package:
   dotnet add package HotChocolate.AspNetCore

2. Create a GraphQL schema class:
   public class Query
   {
       public string Hello() => "Hello, World!";
   }

3. Configure GraphQL in Startup.cs:
   public void ConfigureServices(IServiceCollection services)
   {
       services.AddGraphQL(sp => SchemaBuilder.New().AddQueryType());
   }

4. Add the GraphQL endpoint in Configure method:
   app.UseRouting();
   app.UseEndpoints(endpoints =>
   {
       endpoints.MapGraphQL();
   });

Schema-First vs Code-First

With HotChocolate, you can choose between schema-first and code-first approaches:

Schema-First: You define the schema using GraphQL SDL (Schema Definition Language) and implement resolvers.

Code-First: You define your schema using C# code and rely on reflection to create the schema.

Example Query

Once your GraphQL API is set up, you can query data like this:


query {
   hello
}

This would return:


{
  "data": {
    "hello": "Hello, World!"
  }
}
