SQL Server Interview Guide

SQL Server Indexing - Questions and Answers

1

What is an index in SQL Server?

An index in SQL Server is a database object that improves the speed of data retrieval operations on a table at the cost of additional space and decreased performance on data modification operations.

2

What are the types of indexes in SQL Server?

The two primary types of indexes in SQL Server are clustered and non-clustered indexes. A clustered index determines the physical order of rows, while non-clustered indexes are separate structures.

3

How do you create an index in SQL Server?

You can create an index using the CREATE INDEX statement.

CREATE INDEX idx_student_name
ON Students(Name);
4

What is a clustered index?

A clustered index sorts and stores the data rows in the table or view based on the index key. A table can only have one clustered index.

CREATE CLUSTERED INDEX idx_student_id
ON Students(ID);
5

What is a non-clustered index?

A non-clustered index creates a separate structure from the data table and points to the physical location of data. A table can have multiple non-clustered indexes.

CREATE NONCLUSTERED INDEX idx_student_name
ON Students(Name);
6

What is a unique index?

A unique index ensures that the values in the indexed column(s) are unique, preventing duplicate values.

CREATE UNIQUE INDEX idx_unique_student_email
ON Students(Email);
7

What is a composite index?

A composite index is an index that includes multiple columns in its key. It helps optimize queries that filter or sort by those columns.

CREATE INDEX idx_student_fullname
ON Students(FirstName, LastName);
8

How does an index improve performance?

Indexes improve the performance of SELECT queries by allowing SQL Server to quickly locate and retrieve the required data without scanning the entire table.

9

What is index fragmentation?

Index fragmentation occurs when the logical order of the index pages does not match the physical order of the pages, leading to inefficient data retrieval.

-- Reorganize fragmented index
ALTER INDEX idx_student_name ON Students REORGANIZE;
