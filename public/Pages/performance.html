SQL Server Interview Guide - Performance Tuning, Error Handling, Partitioning

Performance Tuning

1

What is indexing in SQL Server?

Indexing is a technique used to speed up the retrieval of rows from a table by creating a data structure that allows faster searches.

2

What types of indexes are available in SQL Server?

SQL Server supports several types of indexes, including clustered, non-clustered, and full-text indexes.

3

What is a query execution plan?

A query execution plan is a set of steps SQL Server takes to execute a query. You can view it using SQL Server Management Studio (SSMS) to analyze and optimize query performance.

EXPLAIN SELECT * FROM Users;
4

How do you improve query performance in SQL Server?

You can improve query performance by optimizing indexes, avoiding unnecessary joins, using proper data types, and analyzing execution plans.

Error Handling

5

What is the role of TRY...CATCH in SQL Server?

The TRY...CATCH block is used to handle exceptions in SQL Server. The code inside the TRY block is executed, and if an error occurs, the CATCH block handles it.


BEGIN TRY
    -- Your SQL code here
END TRY
BEGIN CATCH
    -- Error handling code here
END CATCH
6

How do you log errors in SQL Server?

You can log errors using the ERROR_MESSAGE(), ERROR_NUMBER(), and other error functions inside the CATCH block.


BEGIN CATCH
    PRINT ERROR_MESSAGE();
END CATCH
7

What is the difference between RAISEERROR and THROW?

RAISEERROR is used to generate custom error messages, while THROW is used to throw an error that was previously caught.

Partitioning

8

What is table partitioning in SQL Server?

Table partitioning involves splitting a large table into smaller, more manageable pieces, called partitions, based on a partition key.

9

What are the benefits of partitioning?

Partitioning improves query performance, manageability, and helps with archiving data by distributing data across multiple filegroups.

10

How do you create a partitioned table?

You create a partitioned table using the PARTITION SCHEME and PARTITION FUNCTION in SQL Server.


CREATE PARTITION FUNCTION MyPartitionFunction (int)
AS RANGE LEFT FOR VALUES (1, 1000, 2000);

CREATE PARTITION SCHEME MyPartitionScheme
AS PARTITION MyPartitionFunction
TO ([Primary], [Secondary], [Tertiary]);
11

What are some partitioning strategies in SQL Server?

Some partitioning strategies include range, list, and hash partitioning, depending on the type of data distribution needed.
