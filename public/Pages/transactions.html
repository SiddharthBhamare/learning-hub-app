SQL Server Interview Guide - Transactions

SQL Server Transactions - Questions and Answers

1

What is a transaction in SQL Server?

A transaction is a unit of work that is performed on the database. It ensures that all operations within it are completed successfully, and if any operation fails, the transaction is rolled back.

2

What are the ACID properties of a transaction?

The ACID properties ensure that transactions are processed reliably:

Atomicity: All operations in a transaction are treated as a single unit.
Consistency: The database must transition from one valid state to another.
Isolation: Transactions do not interfere with each other.
Durability: Once a transaction is committed, it is permanent.

3

What is the COMMIT command?

The COMMIT command is used to save the changes made during the transaction to the database.

COMMIT;
4

What is the ROLLBACK command?

The ROLLBACK command is used to undo changes made during a transaction.

ROLLBACK;
5

What are the isolation levels in SQL Server?

SQL Server provides several isolation levels to control how transactions are handled:

READ UNCOMMITTED
READ COMMITTED
REPEATABLE READ
SERIALIZABLE

6

How do you begin a transaction in SQL Server?

You can begin a transaction using the BEGIN TRANSACTION

7

What is a savepoint in SQL Server?

A savepoint allows you to set a point within a transaction to which you can roll back, without affecting the entire transaction.

SAVE TRANSACTION savepoint_name;
8

What is the purpose of the SET IMPLICIT_TRANSACTIONS option?

The SET IMPLICIT_TRANSACTIONS option is used to automatically begin a new transaction after each SQL statement is executed.

SET IMPLICIT_TRANSACTIONS ON;
